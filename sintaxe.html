HTML

É um acrônimo para Hyper Text Markup Language. Não é uma linguagem de programação e sim uma linguagem de marcação de hiper texto.

Por ser uma linguagem de marcação, o HTML usa essêncialmente marcações ou mais conhecido como Tags.

Você pode reconhecer as tags por estarem entre os sinais de < (menor que) e > (maior que). A maior parte delas inicia <> e termina </>. Mas nem todas tem o fechamento </>.

    Exemplos de tag com fechamento:
  • <html></html> - Inicia a construção de um documento HTML.
  • <head></head> - Cabeçalho do documento.
  • <body></body> - Corpo do documento.
  • <div></div> - Cria um container, uma caixa onde você pode agrupar elementos.
  • <h1></h1> - Geralmente usado para títulos. Coloca uma formatação no texto como tamanho e negrito sendo h1 o maior tamanho e diminuindo até o h6. Também é usado por buscadores para identificar o título da página.
  • <section></section> - Similar ao div, cria uma seção na página.
  • <strong></strong> - Deixa o texto em negrito. Dá o significado que a palavra tem destaque, força.
    Exemplos de tag sem fechamento:
  • <br> - Insere uma quebra de linha. É uma tag "vazia", usada para forçar uma nova linha sem iniciar um novo parágrafo.
  • <hr> - Cria uma linha horizontal de separação.
  • <link> - Define a relação entre o documento atual e um recurso externo, como uma folha de estilos CSS.
  • <img> - Usada para exibir uma imagem. Precisa dos atributos src (origem da imagem) e alt (texto alternativo).
  • <input> - Cria campos de entrada em formulários, como caixas de texto ou botões.

Elementos HTML

Um elemento é um bloco de construção de uma página web. Enquanto podemos dizer que existe a tag <p>, podemos chamar de elemento quando temos seu uso completo como em: <p>Este é um parágrafo.</p>

  • Tag de abertura: <p>
  • Conteúdo: Este é um parágrafo.
  • Tag de fechamento: </p>

Também existem elementos vazios como é o caso da Tag <img> para inserir uma imagem na página.

Estrutura de uma tag

Como já vimos uma tag HTML é um elemento que marca e organiza o conteúdo de uma página web, informando ao navegador como formatar o texto e exibir elementos como títulos, parágrafos e imagens. Elas funcionam como "etiquetas" que delimitam o início e o fim de uma estrutura, sempre contidas entre os sinais de menor que < e maior que >.

Algumas tags também possuem atributos que fornecem informações adicionais ou modificam o comportamento padrão do elemento. Todo atributo virá após o nome da tag e seu valor será definido após o sinal de igual. <img src="c:\imagem.jpg">, onde src é o atributo e seu valor é c:\imagem.jpg.

Comentários

São um recurso essencial, não para o usuário final, mas para o desenvolvedor. Eles são trechos de texto inseridos no código-fonte que são ignorados e não renderizados pelo navegador, servindo unicamente para fins de documentação interna, organização e depuração.

    Em HTML, os comentários são delimitados pela seguinte sintaxe:
  • O comentário deve começar com <!-- (sinal de menor que, exclamação, dois hífens) e terminar com --> (dois hífens, sinal de maior que).
  • Exemplo: 
        
        <!-- Comentário: criação do elemento título da página -->
        <h1>Título</h1>

Estrutura básica de um documento HTML


    <!DOCTYPE html>
    <html lang="pt-br">
        .
        .
        .
    </html>

Na primeira linha <!DOCTYPE html> é definido o tipo de documento que nesse caso é html.

Já a segunda linha <html lang="pt-br"> é iniciada a tag html com o atributo que define a linguagem que essa página será escrita.

Dentro das tags <html> e </html> vamos usar mais duas que são essênciais, que são, <head> que indica o cabeçalho da página e onde são definidas algumas configurações do documento e <body> que é o corpo da página e onde estará todo seu conteúdo.

E para definir visualmente o que está dentro de que, ou seja, quem faz parte de quem, usamos uma técnica chamada indentação, que nada mais é do que afastar o texto da margem usando a tecla TAB. Isso ajuda a tornar mais legível e facíl de compreender o documento.

Um documento sem indentação ficaria dessa forma:


    <!DOCTYPE html>
    <html lang="pt-br">
    <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Título do documento</>
    </head>
    <body>
    <h1>Olá, Mundo!</h1>
    </body>
    </html>

Então adicionando a técnica, essas tags ficam assim:


    <!DOCTYPE html>
    <html lang="pt-br">
        <head>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <title>Título do documento</>
        </head>
        <body>
            <h1>Olá, Mundo!</h1>
        </body>
    </html>

Visualmente você consegue perceber o que chamamos de agrupamento de tags, que nada mais é do que colocar uma tag dentro de outra. O agrupamento desempenha um papel vital em nossa página da web porque ajuda o desenvolvedor (e os buscadores) a segmentar o conteúdo, o que torna mais fácil de posicionar, estilizar ou manipular o documento.

Você consegue perceber que adicionamos mais do que apenas o <head> e <body>. Então vamos a explicação:

  • <meta> - As meta tags são metadados que auxiliam mecânismos de busca e configurações do documento. São dados que fornecem informações sobre o documento HTML, mas que não são exibidos diretamente na página e sempre virão dentro da tag <head>.

    Nosso primeiro meta na linha 4 usa o atributo charset que indica o conjunto de caracteres que será usado, neste caso o UTF-8 que é responsável por caracteres com acento e ç.

    O segundo meta possui dois atributos, name responsável por identificar o tipo de metadado e content que é o valor ou conteúdo do metadado.

    Em html5 existem nomes padrão de metadados, alguns deles são:

    • viewport - Indica o valor inicial do viewport (área visível da janela do navegador onde o conteúdo da página é exibido). É comum em sites responsivos, que são construidos para serem legíveis independente do dispositivo que o acesse. Seu conteúdo mais usado é: width=device-width, initial-scale=1.0 que possuem dois atributos que são width: que define a largura do documento e tem o valor device-width que significa largura do dispositivo. O outro atributo initial-scale: representa o zoom inicial quando a página é carregada pela primeira vez. O valor 1.0 é o nível de zoom normal.
    • application-name - Define o nome do aplicativo web em execução. Se a página não for um aplicativo web, este nome de metadados não deve ser usado.
    • author - Define o nome do autor do documento.
    • description - Contém um resumo curto e preciso do conteúdo da página.
    • keywords - Define palavras-chave separadas por vírgula associadas ao conteúdo da página onde os buscadores usam para associar sua página a busca dos usuários.
    • publisher - Define o nome do editor, a pessoa que cria/edita os textos da página.
  • <title> - Define um título para o documento. Este aparece nas abas ou barra de título dos navegadores.
